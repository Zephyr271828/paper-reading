# TAGS

Canonical tag registry for paper cards in this vault.

Use these tags in both places on each card:
- YAML frontmatter, for example `tags: [DLM, SPEC_DECODING]`
- Inline hashtags in the body, for example `#DLM #SPEC_DECODING`

Do not create or update per-tag note files under `tags/`. Obsidian's native tag filtering is the source of truth for tag-based browsing.

| Tag | Meaning |
|-----|---------|
| `DLM` | Discrete / diffusion language models |
| `RL` | Reinforcement learning for LLMs |
| `SPEC_DECODING` | Speculative decoding / inference acceleration |
| `MLSYS` | ML systems (memory, kernels, scheduling) |
| `VISION` | Computer vision / visual representation learning / vision foundation models |
| `PRUNING` | Pruning and sparsity |

## Adding New Tags

Add a new tag only when it captures a coherent, recurring research area that multiple papers could share. Keep tags canonical, uppercase, and reusable. Good examples include tags such as `QUANT` for quantization or `ARCH` for novel architecture design. Do not create one-off tags for a single paper.
